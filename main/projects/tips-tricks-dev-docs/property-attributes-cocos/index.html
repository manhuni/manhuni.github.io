🎮 Decorators trong Cocos Creator v3

Tổng hợp các decorator phổ biến trong Cocos Creator như @property, @requireComponent, @disallowMultiple... giúp bạn viết mã rõ ràng, dễ hiểu và tương thích tốt với Editor.

📌 Mục tiêu

  • ✔️ Giải thích ý nghĩa và cách dùng của từng decorator
  • ✔️ Cung cấp ví dụ thực tế dễ áp dụng
  • ✔️ Tối ưu hóa trải nghiệm khi làm việc với Editor

💡 Một số decorator tiêu biểu

  • @property: Cho phép hiển thị trường dữ liệu trong Inspector.
  • @requireComponent: Tự động thêm component phụ thuộc.
  • @disallowMultiple: Không cho phép gắn nhiều instance lên cùng node.
  • @executeInEditMode: Cho phép chạy script trong Edit Mode.
  • @visible: Hiển thị có điều kiện.
  • @menu: Gắn menu cho Component.
  • @help: Gắn link tài liệu vào dấu hỏi trên Inspector.

🧪 Ví dụ thực tế

import { _decorator, Component, Label, Collider2D } from 'cc';
const { ccclass, property, requireComponent, disallowMultiple, executeInEditMode, menu, help } = _decorator;

@ccclass('GameManager')
@requireComponent(Collider2D)
@disallowMultiple()
@executeInEditMode
@menu('Game/Managers/GameManager')
@help('https://yourgame.dev/docs/GameManager')
export class GameManager extends Component {
  @property({ type: Label, tooltip: "Hiển thị điểm số" })
  scoreLabel: Label = null;

  @property
  private enableDebug: boolean = false;

  @property({ visible() { return this.enableDebug; } })
  debugInfo: string = "";
}

🔍 Giải thích chi tiết

  • @property: Cho phép kéo thả Label trong Editor.
  • visible(): Dựa vào biến `enableDebug` để ẩn/hiện `debugInfo`.
  • @requireComponent(Collider2D): Tự thêm Collider2D nếu chưa có.
  • @menu: Hiển thị thành “Game → Managers → GameManager”.
